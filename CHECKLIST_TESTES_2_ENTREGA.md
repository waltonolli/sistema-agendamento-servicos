# Checklist de Testes – 2ª Entrega

Use este checklist antes de gravar o vídeo e enviar a entrega.

## 1. Preparação

- [ ] Repositório atualizado no GitHub.
- [ ] Backend com dependências instaladas.
- [ ] Frontend com dependências instaladas.
- [ ] Arquivo `.env` criado no Backend.
- [ ] Arquivo `.env` criado no Frontend, se necessário.
- [ ] Backend rodando sem erro.
- [ ] Frontend rodando sem erro.

## 2. Teste do backend

- [ ] Servidor inicia na porta 5000.
- [ ] Banco SQLite é criado/sincronizado.
- [ ] Rotas de autenticação respondem.
- [ ] Rotas de agendamento respondem.
- [ ] Erros aparecem em formato JSON.

## 3. Teste do cliente

- [ ] Cadastro de cliente funciona.
- [ ] Login de cliente funciona.
- [ ] Cliente visualiza prestadores disponíveis.
- [ ] Cliente cria solicitação de agendamento.
- [ ] Cliente visualiza status pendente.
- [ ] Cliente consegue acompanhar histórico.

## 4. Teste do prestador

- [ ] Cadastro de prestador funciona.
- [ ] Login de prestador funciona.
- [ ] Prestador visualiza solicitações.
- [ ] Prestador aprova agendamento.
- [ ] Prestador rejeita agendamento.
- [ ] Status do agendamento muda corretamente.

## 5. Evidências para entrega

- [ ] Print do repositório GitHub.
- [ ] Print do backend rodando.
- [ ] Print do frontend rodando.
- [ ] Print do cadastro/login.
- [ ] Print do agendamento criado.
- [ ] Print do agendamento pendente.
- [ ] Print do agendamento aprovado ou rejeitado.

## 6. Antes de gravar o vídeo

- [ ] Deixar backend aberto no terminal.
- [ ] Deixar frontend aberto no navegador.
- [ ] Criar usuário cliente e prestador de teste.
- [ ] Testar o fluxo completo uma vez.
- [ ] Fechar abas desnecessárias.
- [ ] Deixar o roteiro aberto para consulta.
